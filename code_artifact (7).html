الحدود الزمنية وانتهاء الجلسة (Time Limits)

الدرس السادس: منح المستخدمين الوقت الكافي والمتحكم به لإتمام المهام

⏰

1. المشكلة: الخروج المفاجئ

تقوم بعض المواقع الإلكترونية والخدمات المصرفية بتسجيل خروج المستخدمين تلقائياً لحماية الأمان دون مراعاة سرعة إدخال البيانات:

  • ضياع مجهود الإدخال: عند انتهاء صلاحية الجلسة فجأة، يفقد المستخدم كل البيانات الحساسة التي قام بملئها في النماذج الطويلة.
  • عدم وجود تحذير مسبق: يخرج النظام المستخدم بقرار مفاجئ وصامت دون منحه خيار طلب تمديد الوقت.
مثال خاطئ بالصورة: انتهاء صلاحية الجلسة وتوقفها بعد 5 دقائق تماماً دون إصدار أي تنبيه مرئي أو مسموع للمستخدم.
♿

من يتأثر بهذا العائق؟

تحديد الوقت بشكل صارم ومباغت يؤثر بشكل مباشر وعنيف على الفئات التالية:

👴 كبار السن (Older Adults):

قد يواجهون صعوبة في الكتابة السريعة أو يحتاجون وقتاً أكبر لمراجعة الحقول وقراءتها بتمهل.

🧠 ذوو الصعوبات الإدراكية:

الذين يحتاجون لقراءة الإرشادات بتركيز مكرر دون ضغوط عامل الوقت المتسارع.

🖥️ مستخدمو التقنيات المساعدة:

مستخدمو قارئات الشاشة والتحكم بالصوت يستغرقون وقتاً أطول للتنقل بين الحقول وتعبئتها حركياً.

✨

الممارسة السليمة والحل

تنص معايير الإتاحة العالمية على منح المستخدمين الوقت الكافي لإكمال المهام عبر توفير الخصائص التالية:

✔ ظهور نافذة تنبيه: إبراز رسالة تحذير واضحة تفيد بقرب انتهاء الوقت (مثال: "ستنتهي جلستك خلال دقيقتين").
✔ خيار تمديد الوقت: توفير زر مباشر لطلب تمديد الجلسة بنقرة واحدة والحفاظ على البيانات.
✔ التحكم المرن: إمكانية تعطيل المؤقت تماماً أو تخصيص مدة أطول للجلسة من البداية.
🎯 المبدأ التوجيهي الرئيسي: "يجب أن يملك المستخدم السيطرة والقدرة الكاملة على تمديد الوقت لإنهاء مهامه بأريحية وأمان بوجب المعيار WCAG 2.2.1."
❌ ممارسة خاطئة ومقلقة No Warning Timeout

بوابة الدفع وبطاقات الائتمان

في هذا النموذج، يبدأ مستخدم كفيف أو كبير في السن بإدخال رقم بطاقته الطويل وحساباته المصرفية بدقة بالغة وبطء تام. سينتهي الوقت فجأة دون تلميح مسبق!

🕒 وقت الجلسة الإجمالي للمحاكاة: 10 ثوانٍ
⛔

انتهت صلاحية الجلسة!

تم تسجيل خروجك تلقائياً وفقدت كافة البيانات التي قمت بإدخالها بسبب انتهاء الوقت المسموح (5 دقائق).

✅ ممارسة سليمة (متاحة تماماً) Accessible Session Control

بوابة الدفع والبطاقات الآمنة والولوجة

قبل انتهاء الوقت المحدد بنصف المدة، يقوم النظام بفتح نافذة منبثقة تفاعلية تعلن بالصوت والصورة عن قرب انتهاء الجلسة، مع منح المستخدم خياراً سهلاً للتمديد.

🕒 وقت الجلسة الإجمالي للمحاكاة: 10 ثوانٍ
🚪

تم تسجيل خروجك بأمان

انتهى الوقت وتم حفظ بياناتك بشكل آمن دون تسريبها، يمكنك العودة وتسجيل الدخول متى شئت.

⚠️

تنبيه: ستنتهي صلاحية جلستك قريباً!

حفاظاً على أمان بياناتك، ستنتهي جلستك خلال 2 دقيقة (محاكاة: 5 ثوانٍ متبقية). هل ترغب في تمديد الجلسة؟

شاشة المطورين والمحاكاة التفاعلية (Live Screen Reader Voice Sim)

شاهد واستمع إلى ما تنطق به التقنيات المساعدة للمستخدم الكفيف في اللحظات الحرجة لانتهاء وقت الاستمارة:

// اضغط على أحد أزرار المحاكاة في الأعلى لبدء متابعة خط سير تصفح الكفيف...
* يتم نطق العبارات صوتياً باللغة العربية عبر محرك التركيب الصوتي للمتصفح عند تفعيل النطق الصوتي بالأعلى.

الشرح البرمجي للمطورين

كيف يمكننا برمجة مؤقتات تفاعلية مريحة ومتاحة لجميع الأدوات المساعدة؟

❌ الكود الصامت والمفاجئ (الخاطئ)

تسجيل خروج المستخدم فجأة دون إشعاره، مما يؤدي لمسح البيانات عشوائياً.

// كود صامت تماماً يفاجئ الكفيف بموت الجلسة فجأة
setTimeout(function() {
  // انتهاء الخمس دقائق ومسح البيانات
  logoutUser(); 
  showLoggedOutScreen(); 
}, 300000); // 5 دقائق صامتة تماماً

✅ كود التحذير وإدارة التركيز (الصحيح)

تنبيه المستخدم قبل الوقت الكافي مع إعطائه صلاحية التحكم وحبس التركيز بالنافذة.

// تنبيه المستخدم قبل دقيقتين من النهاية
setTimeout(function() {
  // 1. فتح نافذة التنبيه المنبثقة
  showWarningModal();
  
  // 2. تحديث الـ ARIA ونطق التحذير لقارئ الشاشة
  announceToScreenReader("تحذير: ستنتهي جلستك خلال دقيقتين. هل تود التمديد؟");
  
  // 3. توجيه التركيز لزر "تمديد الوقت" لسهولة الوصول بنقرة واحدة
  document.getElementById('extendBtn').focus();
}, 180000); // إظهار التنبيه بعد 3 دقائق من أصل 5
💡 معلومة برمجية هامة للشرح: عند فتح نافذة تمديد الوقت، يجب إتاحة إغلاقها ومغادرتها بضغطة زر الـ Escape، والحرص على أن يكون زر "تمديد الوقت" هو العنصر الأول الذي يستقبل التركيز مباشرة (Auto Focus) لمنع ضياع مستخدمي لوحة المفاتيح.

تم التطوير كأداة تفاعلية حية لشرح ومحاكاة معايير سهولة الوصول للحدود الزمنية (Digital Accessibility WCAG 2.2.1 Time Limits and Session Timeouts).
